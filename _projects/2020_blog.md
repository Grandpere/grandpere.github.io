Ce projet a été réalisé seul pas vraiment en condition réelle puisque j'étais confiné avec mon enfant que je ne pouvais laisser seul. Le but était de réaliser un "blog communautaire" du style medium/dev.to afin de mettre en pratique mes compétences sur Symfony et d'en apprendre d'autre.

J'étais initialement parti pour développer une API pouvant servir sur d'autres supports mais, bien que cette idée soit la meilleure pour le futur de l'application, n'était pas le plus pertinent pour fournir le MVP que je souhaitais.

J'ai donc repris le développement d'une application classique sans API. J'ai dans un premier temps conçu un MCD et quelques users stories afin de documenter les fonctionnalités et la conception de l'application.

J'ai implémenté :
- l'upload de fichier via VichUploaderBundle que j'ai experimenté par la même occasion
- la récupération très basique d'un avatar via Gravatar
- le parcours complet d'un utilisateur (inscription, connexion, activation de compte, mot de passe oublié, etc.)
- les fixtures via un bundle que je n'avais pas encore testé jusqu’à présent.
- la fonctionnalité impersonate User utilisé dans des backend comme Sonata pour visualiser directement une application selon le profil de l'utilisateur.
- un voter et j'ai approfondi mes connaissances sur les voters de Symfony
- des requêtes personnalisées dans mes repository.
- un dataTransformer
- autocomplétion ajax des catégories d'articles.
- un formulaire de contact
- des tests unitaires et fonctionnels basiques via PHPUnit (non fini).

J'ai aussi testé des outils d'intégration continue comme TravisCi / CircleCi, comment réduire le temps de réponse de mes pages par plusieurs moyens (optimisation des requêtes, compilation des assets via webpackencore, mise en cache).

Le projet n'est pas terminé, j'avance en fonction de ma recherche d'emploi, événement emploi, etc.
